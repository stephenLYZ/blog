Web Security - XSS

08-22

一直没有机会去了解web安全，直到写chatroom应用的时候才发现了这个严重的问题（导致后来闹了点笑话），于是下定决心开始学习web安全的知识，以下是我的整理。

web安全经典的攻击方式主要分为三种: XSS、CSRF、界面操作劫持，本文主要说的是XSS，这篇文章也只是XSS攻击的冰山一角，如有不对的地方望指出。

什么是XSS

XSS(Cross Site Scripting) 称为跨站脚本攻击，是一种经常出现在web应用中的计算机安全漏洞，它允许攻击者将脚本（比如javascript）嵌入到客户端中，当用户浏览网页的时候，脚本就会在用户的浏览器上执行，这也就意味着攻击者可以通过脚本获取用户的Cookie，导航到恶意网站，携带木马等等。

XSS类型

Stored XSS

Stored XSS ，攻击者将脚本注入到web服务器上，由于这种攻击类型具有很强的稳定性，危害也比较大，所以又叫Persistent XSS。举个简单的例子，假设一个博客网站存在XSS漏洞，攻击者发布一篇文章，其中嵌入javascript代码，这篇文章保存在数据库中，其他人如果访问这篇文章，javascript代码就会在访问者的浏览器中执行，那么cookie或者其他信息就很有可能会被攻击者盗走。
